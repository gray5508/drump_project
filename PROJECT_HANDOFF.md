# 谱面练习台项目交接说明

更新时间：2026-07-22

这份文件是后续新任务的主要入口。开始修改前，应先阅读本文件，再检查实际代码和 Git 状态。聊天记录只作为补充，不应当作为唯一依据。

## 1. 当前目标

这是一个面向个人架子鼓练习的谱面工具，核心体验是：

- 正常显示整张鼓谱，同时可以按小节悬停、标色和选中。
- 支持完整谱面、按行练习、小节编排三个界面。
- 每个小节可以绑定教学视频或伴奏时间段。
- 通过“麦当劳”唤醒语音控制，练鼓时尽量减少触屏操作。
- 目前同时维护离线文件夹和 GitHub Pages 公网版本。
- 后续考虑增加局域网本地部署，但尚未正式实施。

## 4. 三个练习界面

### 完整谱面

- 显示整张鼓谱和小节覆盖层。
- 悬停显示淡色预览和小节编号。
- 单击使用当前颜色标记；再次使用相同颜色单击可取消。
- 支持缩放。
- 语音“第 N 小节”会清除其他标记，只保留并定位目标小节。

### 按行练习

- 每次显示一行谱面。
- 支持上一行、下一行、数字跳转和键盘左右方向键。
- 颜色标记与完整谱面共享。
- 语音定位小节后会跳到对应行并标记目标小节。

### 小节编排

- 默认读取当前已标记小节，每行最多四个。
- 支持拖动排序、跨行移动、保存命名组合和读取组合。
- 模板框统一控制默认尺寸，每个小节框也可四角单独缩放。
- 单独尺寸会随组合保存。
- 读取其他组合前，如果当前内容未保存，会询问保存、丢弃或取消。
- 当前界面只开放部分语音操作；谱面定位和上下小节等操作会被限制。

## 5. 教学视频

视频登记在 `app.js` 的 `VIDEO_LIBRARY`。

当前登记的小节包括：40、41、46、48。

约定：

- 视频放在 `video/`。
- 海报放在 `video/posters/`。
- 单片段可命名为 `measure-小节号.mp4`。
- 多片段使用 `measure-小节号-1.mp4`、`measure-小节号-2.mp4`。
- 视频窗口下方同步显示对应小节谱面，并可调节谱面大小。
- 有视频的小节底部显示红线。
- 视频窗口也显示语音唤醒水纹和状态。

视频语音状态：

- “暂停”暂停当前视频。
- “播放”在视频已打开时继续视频。
- “重新播放 / 重播”将视频跳回开头并播放。
- “退出 / 关闭 / 返回谱面”关闭视频窗口。
- 视频打开时，上一个/下一个相关指令优先切换教学视频。

## 6. 伴奏播放器

当前音频：

```text
assets/就让这大雨全都落下.mp3
assets/就让这大雨全都落下[rebalanced].mp3
```

两份音频长度一致，第二份为突出鼓声版本。

播放器标题“练习伴奏 ⌄”本身是曲目菜单入口：

- 原版伴奏
- 突出鼓声

切换曲目时需要保持：

- 当前播放位置
- 当前倍速
- 当前播放/暂停状态
- 当前小节试听范围

当前选择保存在本机浏览器中。

播放器还支持：

- 唱片或按钮播放/暂停。
- 常驻进度条和片段相对进度条。
- 0.50～2.00 倍速并尽量保持原音高。
- 长按播放器空白区域后拖动进行精确进度调整。
- 展开、最小化和拖动位置保存。
- 小节试听时显示独立的开始线、结束线和退出试听按钮。

## 7. 小节伴奏时间与长按默认动作

双击小节打开伴奏时间设置，可保存：

- 开始时间
- 结束时间（可到歌曲结尾）
- 该小节倍速
- 长按默认动作：`video` 或 `backing`

时间始终按原曲 1 倍速时间轴保存；改变播放倍速不修改保存的时间点。

默认动作规则：

- 已设置 `holdAction: "backing"`：长按或默认“播放”会试听该小节伴奏。
- 其他情况：默认打开教学视频。
- 默认是视频但没有视频时，明确提示缺少视频，不自动改播伴奏。

以下行为必须调用同一个默认动作函数，避免逻辑分叉：

- 用户真实长按某小节。
- 语音“播放第 N 小节”。
- 当前仅选中一个小节时说“播放”或“长按”。

强制伴奏指令：

- “试听第 N 小节”
- “播放第 N 小节伴奏”
- “试听当前小节”

小节底部指示：

- 蓝线：有伴奏时间。
- 红线：有教学视频。
- 同时存在时显示两条线。

## 8. 语音架构

当前语音分两阶段：

1. 浏览器 Web Speech API 低成本等待唤醒词“麦当劳”。
2. 唤醒成功后连接千问实时语音，采集平板麦克风并识别正式指令。

关键状态：

- 初始只等待“麦当劳”，尚未连接千问。
- 唤醒后获得完整 10 秒指令窗口。
- 每次成功匹配指令后，重新获得完整 10 秒。
- 窗口结束、连接失败或连接关闭后，清理当前云端状态并返回唤醒等待。
- 唤醒时全页面显示低透明度扩散水纹；视频窗口中也有对应反馈。
- 页面上三分之一处短暂显示“麦当劳”之后的识别内容。

千问代理地址当前配置在 `qwen-cloud.js`：

```text
https://drump-qrealtime-cxfvcbehsz.cn-beijing.fcapp.run
```

页面先访问 `/health`，再连接 `/ws/realtime`。API Key 必须保留在阿里云函数环境变量中，禁止写入网页、交接文档或 Git 仓库。

### 热词和纠错

- 千问识别上下文来自 `VoicePractice.hotwordCorpus`。
- “试听”在上下文中被重复并优先强调。
- 本地将“视听、试停、试厅、时听、十听、十厅”等归一为“试听”。
- 唤醒词仅保留“麦当劳”，并兼容部分常见识别误差。

### 当前上下文媒体指令

| 指令 | 逻辑 |
| --- | --- |
| `播放` / `长按` | 视频已打开则继续视频；小节试听仍激活则继续片段；否则执行当前唯一选中小节的长按默认动作 |
| `继续播放` / `恢复播放` | 视频已打开则继续视频，否则继续伴奏 |
| `暂停` | 视频已打开则暂停视频，否则暂停伴奏 |
| `重新播放` / `重播` | 视频回到开头；或小节试听回到片段起点 |
| `退出` / `关闭` / `返回谱面` | 优先关闭帮助，其次关闭视频，再次结束小节试听 |
| `播放第 N 小节` | 执行该小节长按默认动作 |
| `试听第 N 小节` / `播放第 N 小节伴奏` | 强制播放该小节伴奏片段 |
| `下一段伴奏` / `上一段伴奏` | 切换到相邻的“已配置伴奏时间”的小节，跳过未配置小节 |

已经取消伴奏语音“快进 N 秒 / 后退 N 秒”；手动进度条和长按拖动仍然保留。

注意：`使用说明.txt` 目前仍残留部分旧语音描述，后续应按本节统一更新。

## 9. 数据保存

浏览器本机 `localStorage` 键：

| 键 | 内容 |
| --- | --- |
| `drum-focus-marks` | 小节颜色标记 |
| `drum-focus-groups-v1` | 小节编排组合和单独尺寸 |
| `drum-focus-default-card-height` | 旧版默认高度兼容值 |
| `drum-focus-template-size-v1` | 编排模板宽高 |
| `drum-focus-backing-rate-v1` | 当前伴奏倍速 |
| `drum-focus-backing-segments-v1` | 本机小节伴奏配置 |
| `drum-focus-backing-player-ui-v2` | 播放器折叠状态和位置 |
| `drum-focus-backing-track-v1` | 当前伴奏版本 |
| `drum-focus-video-score-scale-v1` | 视频页小节谱面缩放 |

跨设备规则：

- 标记、组合和播放器偏好默认只保存在当前浏览器，不会自动同步。
- `backing-config.js` 是小节伴奏参数的仓库统一配置。
- 本机已经存在 `drum-focus-backing-segments-v1` 时，本机值优先。
- “恢复仓库配置”会清除本机伴奏参数并重新使用 `backing-config.js`。
- “导出统一配置”会下载新的 `backing-config.js`；替换正式目录文件并发布后，其他没有本机覆盖的设备可读取。

## 11. 离线、联网与局域网方案

当前能力：

- 谱面、标记、编排、音频和视频可作为静态文件运行。
- 千问语音识别需要互联网。
- GitHub Pages 提供 HTTPS，平板麦克风权限相对稳定。

计划中的局域网方案尚未实施。推荐方向：

- 在固定电脑、迷你主机或树莓派上运行一个 Node.js 服务。
- 同一个服务提供静态网站并代理千问 WebSocket。
- 平板通过局域网地址访问。
- 为麦克风权限使用本地 HTTPS，而不是普通 `http://192.168.x.x`。
- 给主机设置固定 IP 或局域网名称，并允许防火墙端口。
- 服务端电脑只需保持服务运行，不需要额外打开浏览器页面。

第一阶段可以先做不带语音的 HTTP 局域网测试；第二阶段再补本地 HTTPS 和语音代理。

## 12. 安全边界

- 不得把阿里云或千问 API Key 写入前端、GitHub Pages、交接文档或提交记录。
- API Key 只放在服务端环境变量中。
- 不要通过“前端加密、前端解密”保护 API Key；解密逻辑和密文同时在网页中等同于暴露。
- 修改阿里云函数前先确认用户授权；仅修改静态练习台时不要顺带更改云函数。
- 不要恢复或覆盖工作区中用户已有的无关删除和未跟踪目录。

## 14. 后续修改原则
